Configuring a Client to Point to a Network Server

In a system with a network server, licenses will be located on the network server and will be shared with all clients located on the network.
To connect to the network server, the network client must be configured so that the license manager can find the network server
There are currently 2 ways to configure the network client 1) through the compiler installer and 2) via the command line.

Configuration through the Installer

When running the installer choose the network client option as shown below

Next enter the properties of the network server. See your network administrator to acquire the correct Server Name and Port Number values.

Configuration via Command Line

To configure a network client using the command line, execute:
> xclm -netfile <port> <host>

The port is a server port.
The host is the IP address on the domain name of the server machine.

See your network administrator to acquire valid port and host values.

Testing to see if Configuration Worked

To test if the client configuration worked execute the command:
> xclm -status

You should now see the products that you purchased, like the example below:

 
> xclm -status                                 
Node locked licenses available:
    Product:rlm_roam Version:1.0 Key:
Network licenses available:
    Product:swxc32n-cpp Version:1.1 Count:1 Key:0038-0194-0113-5559

The rlm_roam license is an advanced feature that comes enabled with all compilers.
What we should see is licenses listed under Network licenses other than rlm_roam. If your status is similar to what is shown above then the configuration is complete! No other actions are required.

The compiler is now set to automatically check out licenses every time it is ran
