Command-Line Options Reference

If you wish to control the license manager beyond the installer screens, you can use the following commands to perform the actions listed in that section. Execute XCLM on the command line as:

> xclm [options]

-?, -help

This option displays available options.

-status

This option displays information about all the licenses that are visible to the license manager.

-actkey activation-key [ hostid=mac [ hostname=host ] ]

This option pings the server with the activation-key specified to license your XC compiler.

The optional attributes are meant to be used in systems where the computer that will host the license cannot access the internet. By default, the activation process will grab host information from the current computer and use that to populate fields in the license file. By specifying the optional attributes, you are instructing the activation process to not use the default information to populate the license file. For node-locked licenses, only the optional hostid needs to be specified. For networks, the optional hostid and hostname must be specified.

mac must be a valid MAC address of the computer that will host the activated license.

host must be the valid username of the computer that will host the activated network license.

In order to find out what the correct values are for hostid and hostname, run:

> xclm -hostinfo

which will print out the mac and host values.

-demostat [ product ]

This option displays the demo status for all possible demos (xc8, xc16, xc32). It will display whether a demo license is present in the license folder or not. If the license does exist, this option will display how many days until the demo expires for an active demo license or will display that the license has expired.

If product is specified, the status will be shown for only that product. The product name must be one of the following:

  • xc8
  • xc16
  • xc32

-makedemo product

This option will activate a 60 day demo license for the product specified. The product must be one of the following:

  • xc8
  • xc16
  • xc32

Creating a demo license can only be done once for each product per computer. Rerunning this command for the same computer and product will download the same license that was previously activated.

-netfile port host

This option configures a client to point to a network server with a floating license.

The port is a server port.

The host is the IP address on the domain name of the server machine. Host values can be an IP address like 127.0.0.1 or a named address like employee1.company.com.

See your network administrator to acquire valid port and host values.

-hostinfo

This option displays hosting information for the current computer. The output can be used for the hostid and hostname parameters of the -actkey option.

-roamout product

Network users can permanently checkout a network license (say if they want to go home). The license server should ‘checkout’ and give a license to the client that has executed this option. The license server now has one less license of product that it can give out.
Products supported include the value xc8, xc16, and xc32

It will stay checked out until the user uses the roamin option.

-roamin product

After a call to roamout, users call roamin to check back in the license to the network. The amount of licenses available for the server to check out now increases by one.
Products supported include the value xc8, xc16, and xc32
